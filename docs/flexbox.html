Flex direction

You can decide the direction of your flexbox, if it's going to be a column or a row <x-flex direction=*>

 Available options
  • direction=column
  • direction=column-reverse
  • direction=row
  • direction=row-reverse

Wrap

Specifies whether flex items are forced into a single line or can be wrapped onto multiple lines. If wrapping is allowed, this property also enables you to control the direction in which lines are stacked. wrap=*

 Available options
  • wrap
  • wrap=nowrap
  • wrap=reverse

Visual order

Specifies the order used to lay out flex items in their flex container. Elements are laid out in the ascending order of the order value. Elements with the same order value are laid out in the order in which they appear in the source code. order=*

 Available options
  • order=first
  • order=last
  •  
  • order-s=first
  • order-s=last
  •  
  • order-l=first
  • order-l=last

Justification

Defines how the browser distributes space between and around flex items along the main-axis of their container. jc=*

 Available options
  • jc=start
  • jc=center
  • jc=end
  • jc=between
  • jc=around

Align content

Aligns a flex container's lines within the flex container when there is extra space on the cross-axis. ac=*

 Available options
  • ac=start
  • ac=center
  • ac=end
  • ac=between
  • ac=around
  • ac=stretch

Align items

Aligns flex items of the current flex line the same way as justify-content but in the perpendicular direction. ai=*

 Available options
  • ai=start
  • ai=center
  • ai=end
  • ai=baseline
  • ai=stretch

Align self

To be used on <x-col> Aligns flex items of the current flex line overriding the align-items value. as=*

 Available options
  • as=auto
  • as=start
  • as=center
  • as=end
  • as=baseline
  • as=stretch
